### 1.2 Skill 分層管理（三層 Skill 系統）

- **目標**：引入 core → shared → workspace 三層 Skill 分層機制，提升生命週期管理
- **優先級**：P1（短期～中期）
- **關聯**：improvement-backlog 提案（源自 OpenClaw 302K Stars）、`skills/SKILL_INDEX.md`、`docs/ARCHITECTURE.md`
- **來源靈感**：OpenClaw 的 bundled → managed → workspace 三層 Skill 系統
- **現況分析**：
  - 46 個 Skills 全部扁平管理，無穩定度分層
  - 部分實驗性 Skill（如 skill-forge、workflow-forge）與核心 Skill（todoist、ntfy-notify）同層級
- **建議分層**：
  - **core**（~15 個）：每日摘要必用、高穩定度（todoist、pingtung-news、ntfy-notify、api-cache 等）
  - **shared**（~8 個）：跨專案共用、中等穩定度（knowledge-query、web-research、groq 等）
  - **workspace**（~23 個）：實驗性或低頻使用（skill-forge、workflow-forge、behavior-pattern-analyzer 等）
- **可執行動作**：
  1. 為 `skills/SKILL_INDEX.md` 加入分層標記（`tier: core|shared|workspace`）
  2. 建立 machine-readable 的 `skills/manifest.json`（參考 n-skills 的 openskills 格式），降低文字索引的路由不精確性
  3. 制定分層升降規則：workspace Skill 累積 30+ 次成功執行後可升級為 shared

### 1.3 持久記憶功能 / Agent 及排程長記憶方案

- **目標**：實現跨 Session 的長期記憶，讓 Agent 能累積知識與追蹤長期趨勢
- **優先級**：P1（短期～中期）
- **關聯**：`digest-memory` Skill、`context/continuity/` 連續記憶機制、KB 筆記 `b1640648`（AI Agent 記憶系統研究）、`1b5eb493`（Mem0/Letta/MemGPT 指南）、`72c33b6d`（Memory-Augmented LLM 指南）、`1adc3ffd`（Hindsight 四網路記憶）、`14c8dbd1`（Mastra Observational Memory）
- **現況**：
  - 已有 `digest-memory.json`（跨次摘要記憶）和 `context/continuity/auto-task-*.json`（自動任務連續記憶）
  - 但缺乏**語義記憶整合**（KB 知識與 Agent 決策的橋樑）和**記憶衰減機制**
- **KB 研究基礎（2026-03 最新）**：
  - 三層記憶模型（感覺/短期/長期）已在 KB 有深度研究
  - StateLM（主動記憶管理）、Chain-of-Memory（輕量鏈式記憶）值得借鑑
  - Continuum Memory Architecture（CMA）適合本專案的連續執行模式
  - **Hindsight**（vectorize-io, 4.3K→4.6K Stars）：四網路記憶（Knowledge/Experience/Mental Models/Opinion）+ Retain-Recall-Reflect 三元操作 + MCP 原生整合，LongMemEval 91.4% SOTA
  - **Mastra**（21.8K Stars）：Observational Memory 三層壓縮（Observer 30K tokens → Reflector 40K tokens），LongMemEval 94.87%
  - **Mem0** 效能數據：26% 準確率提升、91% 延遲降低、90% Token 節省
- **ROI 評估框架**（本次新增）：

  | 方案 | 整合難度 | 本專案適配度 | 維護成本 |
  |------|---------|-------------|---------|
  | 現有 JSON 方案 | 已完成 | 高（自建） | 低 |
  | Mem0/Mem0ᵍ | 中等 | 高（跨 Session） | 中 |
  | Hindsight + MCP | 中等 | 高（MCP 原生） | 低 |
  | Mastra Observer | 高 | 中（TypeScript） | 中 |
  | MemGPT/Letta | 高 | 中（需 Docker） | 高 |

- **可執行動作**：
  1. 設計「記憶整合層」：自動將 `continuity/*.json` 的重複模式提煉到 KB 筆記（類似 Memory Consolidation）
  2. 為 `digest-memory.json` 加入趨勢摘要欄位（7 日/30 日滑動視窗）
  3. **優先評估 Hindsight + MCP**：因 MCP 原生整合與本專案架構最契合，且不需 Docker
  4. 建立記憶品質指標：recall accuracy、freshness score、dedup ratio

  1. 參考 pre-commit 的 `.pre-commit-config.yaml` 格式設計 hook registry
  2. 支援 `enabled: true/false`、`stage: pre_bash|pre_write|pre_read`、`order: number`
  3. 統一由 `hook_pipeline.py` 讀取 registry 執行（已有 DeerFlow 式短路機制基礎）
