# 筆記方法論：知識百科筆記法 × 概念圖組織法

> 本文件定義 Knowledge Domain Builder 生成筆記的方法論基礎。
> 本工具的目標是**建立完整的知識參考庫**，不是學習工具或空殼筆記本。
> 每一則筆記都必須是一篇獨立完整的知識文章，讀者打開即可獲得該概念的全貌。

---

## 一、知識百科筆記法（Encyclopedic Note Method）

### 1.1 核心理念

知識百科筆記法以維基百科的條目品質標準為參照——每一則筆記就是一篇**自足的知識文章**（self-contained article）。它不是筆記碎片、不是學習提示卡、不是待填寫的框架，而是一篇經過完整研究、結構化撰寫、可直接引用的知識條目。

與學習導向的筆記法（如原子筆記、費曼學習法、漸進式摘要）的根本差異：

| 面向 | 學習導向筆記 | 知識百科筆記（本系統） |
|------|------------|-------------------|
| 目的 | 幫助使用者內化知識 | 建立完整的知識參考庫 |
| 內容完整度 | 留白讓使用者填寫 | AI 自動填入完整內容，零留白 |
| 篇幅 | 短小精悍（200-500 字） | 深入完整（1500 字以上） |
| 使用者角色 | 主動撰寫者 | 查閱者與策展者 |
| 品質標準 | 個人理解即可 | 百科全書等級的準確與完整 |

### 1.2 六項品質標準

每一則知識筆記必須滿足以下品質標準：

#### 標準一：自足性（Self-containment）

讀者不需要先閱讀其他筆記就能理解這則筆記。筆記內部應包含足夠的背景脈絡——如果引用了其他概念，應在正文中簡要說明，再透過 wikilink 提供深入閱讀的路徑。

#### 標準二：結構完整性（Structural Completeness）

每則筆記必須包含完整的知識結構：定義與概述、詳細說明、具體實例、常見誤解辨析、與相關概念的關係。任何一個區塊缺失都代表筆記不完整。

#### 標準三：深度充分性（Sufficient Depth）

每則筆記的實質內容不得少於 1500 字。這不是灌水，而是確保概念的定義、原理、脈絡、應用、辨析都有充分展開。一個值得獨立成篇的概念，1500 字是描述其全貌的最低門檻。

#### 標準四：準確性（Accuracy）

內容必須基於可靠來源，專業術語的使用必須準確。如涉及引用經典原文或權威論述，應標明出處。

#### 標準五：可查閱性（Retrievability）

筆記的結構應便於快速查閱——清晰的標題層級、重點粗體標記、表格化的比較資訊。讀者應能在 30 秒內找到所需的特定資訊。

#### 標準六：連結完整性（Link Integrity）

每則筆記應透過 wikilink 連結到相關概念，並在正文中說明連結理由。連結不是裝飾，而是知識網路的經緯線。

### 1.3 筆記結構規範

每則知識筆記的標準結構：

```
# 概念名稱

## 概述
200-300 字的完整概述，涵蓋定義、重要性、在所屬體系中的位置。

## 詳細說明
概念的完整闡述，包含：
- 核心原理或義理
- 歷史背景或來源脈絡
- 內部的分類或層次結構
- 運作機制或實踐方法

## 關鍵要點
3-5 個最重要的要點，每個要點配以簡要說明。

## 具體實例
至少一個詳細的實例或應用場景，展示概念如何在實際中呈現。

## 常見誤解辨析
至少一個常見誤解，包含「誤解內容→正確理解→為何容易混淆」。

## 比較與辨析
與相近概念的系統性比較（表格或對比說明）。

## 相關概念
帶有連結理由的 wikilink 列表。
```

## 二、概念圖組織法（Concept Map Organization）

### 2.1 核心理念

概念圖組織法由 Joseph Novak 於 1972 年提出，其理論基礎是 David Ausubel 的有意義學習理論——新知識必須與既有知識結構產生連結才能被有效保存和提取。

在本系統中，概念圖組織法不是用來「學習」，而是用來**組織知識條目之間的關係結構**：

- **階層關係**：上位概念包含下位概念（如「化法四教」包含「藏教」「通教」「別教」「圓教」）
- **橫向關係**：同層概念之間的比較、對比、互補（如「析空觀」vs「體空觀」）
- **跨域關係**：不同分類下的概念之間的連結（如「六即佛位」與「十乘觀法」的對應）

### 2.2 在 Obsidian 中的實現

- **階層關係** → 資料夾結構 + MOC 索引
- **橫向關係** → 筆記正文中的「比較與辨析」區塊
- **跨域關係** → 「相關概念」區塊中帶理由的 wikilink
- **全局視圖** → Mermaid 學習路線圖 + Canvas 知識地圖 + Graph View

### 2.3 MOC 的角色

在知識百科筆記法中，MOC（Map of Content）是**知識索引與導覽頁**，而不是「學習進度追蹤器」：

- 提供該主題下所有條目的概覽表格（附一句話摘要）
- 提供建議的閱讀順序（基於概念依賴關係，非學習進度）
- 包含該主題的概述文字（非空白）
- 使用 Dataview 動態列出相關筆記
