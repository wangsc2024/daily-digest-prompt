# Report Reviewer Skill — 三輪審查優化流程

本 Skill 對已完成的研究報告執行三輪系統性審查，產出優化方案並執行，
確保最終報告在結構完整性、論點品質、寫作表達三個維度都達到最高標準。

---

## 執行流程總覽

```
第一輪審查 → 第二輪審查 → 第三輪審查
      ↓             ↓             ↓
  結構缺口       論點漏洞       語言打磨
      └─────────────┴─────────────┘
                    ↓
            彙整審查意見清單
                    ↓
            擬定優化方案（依優先級排序）
                    ↓
            執行優化（逐項修改）
                    ↓
            輸出最終優化版報告
```

---

## 第一輪審查：結構完整性

**審查對象**：報告的整體架構、章節完整性、研究覆蓋面

審查清單（逐項打勾或標記問題）：

```
[ ] 摘要是否準確涵蓋報告核心發現（非目錄式羅列）？
[ ] 章節是否完整覆蓋研究規劃的所有子議題？
[ ] 各章節是否有清晰的主題句與段落邏輯？
[ ] 章節間的銜接過渡是否自然流暢？
[ ] 結論是否呼應研究目標、整合各章發現而非重複摘要？
[ ] 延伸研究方向是否具體（非泛泛而論）？
[ ] 參考文獻是否完整、格式是否一致？
[ ] 是否有「只有大標題沒有內容」的空洞章節？
[ ] 報告長度是否與主題深度相符（過短 = 淺薄，過長 = 冗贅）？
```

**輸出格式**：
```
【第一輪審查結果】
✅ 通過項目：...
⚠️  問題項目：
  - [章節名稱] 問題描述
  - [章節名稱] 問題描述
缺口評分：_/10（10 = 無缺口）
```

---

## 第二輪審查：論點品質

**審查對象**：論據充分性、邏輯嚴密性、觀點平衡性

審查清單：

```
[ ] 每個核心論點是否有至少一個具體論據支撐（引文/數據/案例）？
[ ] 學術爭議是否呈現多方立場，而非只有單一觀點？
[ ] 是否有「結論跳躍」（論據不足以支撐結論）的情況？
[ ] 是否有「過度概括」（以偏概全）的論述？
[ ] 重要術語是否在首次出現時有清晰定義？
[ ] 引用的文獻資料是否具體（作者、年份、具體說法）？
[ ] 各章節間的論點是否相互呼應，形成完整論述鏈？
[ ] 是否有重複論述同一觀點的冗余內容？
[ ] 是否有未被論據支撐的「主觀判斷」混入學術論述？
```

**輸出格式**：
```
【第二輪審查結果】
✅ 論點紮實項目：...
⚠️  論點問題：
  - [段落位置] 問題描述 → 建議補充方向
  - [段落位置] 問題描述 → 建議補充方向
論點品質評分：_/10
```

---

## 第三輪審查：寫作表達（writing-masters 標準）

**審查對象**：語言清晰度、寫作風格、讀者體驗

參照 `C:\Users\user\.cursor\skills\writing-masters\SKILL.md` 的標準：

審查清單：

```
奧威爾清晰度：
[ ] 是否有可以刪除而不失意義的冗詞贅句？
[ ] 是否過度使用被動語態（應改為主動）？
[ ] 是否使用了不必要的學術行話或艱深詞彙？
[ ] 段落是否過長（超過 250 字應考慮拆分）？

海明威精準度：
[ ] 是否有抽象描述可以用具體細節替換？
[ ] 句子長度是否過於單調（全長句或全短句）？
[ ] 每段首句是否清楚點出段落核心論點？

整體可讀性：
[ ] 標題層級是否清晰合理（H1→H2→H3）？
[ ] 表格、列表的使用是否改善了可讀性？
[ ] 引用區塊是否有效突顯重要引文？
[ ] 整體語氣是否一致（全程學術風格 or 全程平易近人風格）？
```

**輸出格式**：
```
【第三輪審查結果】
✅ 寫作優點：...
⚠️  寫作問題：
  - [位置] 問題描述 → 修改建議
  - [位置] 問題描述 → 修改建議
寫作表達評分：_/10
```

## 彙整與優化方案

三輪審查完成後，執行以下步驟：

### 步驟 A：彙整問題清單

將三輪的所有 ⚠️ 問題彙整，依影響程度分級：

```
【優先級 P1 — 必修（嚴重缺陷）】
  - 影響論點完整性的問題
  - 覆蓋面缺口（重要議題未涵蓋）
  - 邏輯錯誤或論據不足

【優先級 P2 — 應修（品質問題）】
  - 段落過短/過長
  - 引用格式不一致
  - 重複論述

【優先級 P3 — 建議修（提升質感）】
  - 語言優化（主動語態、刪冗詞）
  - 過渡銜接改善
  - 表達精準度提升
```

### 步驟 B：擬定優化方案

針對每個問題，寫出具體修改方案：

```
優化方案格式：
問題：[章節/段落] 具體問題描述
方案：具體修改內容（補充什麼、刪除什麼、改寫哪句話）
預期效果：修改後如何改善報告品質
```

### 步驟 C：執行優化（逐項修改）

按 P1 → P2 → P3 順序逐一執行，每修改一項後記錄：
```
✅ 已修改：[問題摘要] → [修改說明]
```

優化完成後輸出：

```

原始評分：結構 _/10 | 論點 _/10 | 寫作 _/10 | 總分 _/30
優化後評分：結構 _/10 | 論點 _/10 | 寫作 _/10 | 總分 _/30
P1 修改項目：_ 個
P2 修改項目：_ 個
P3 修改項目：_ 個
主要改進：（3-5 條最重要的改進）
```

最終優化版報告覆寫原始 .md 檔案（保留原始版本為 _original.md）。
