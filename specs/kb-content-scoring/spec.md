# KB Content Scoring — 知識庫內容評分系統規格

**版本**：1.0.0
**狀態**：定稿
**建立日期**：2026-03-07

---

## 1. 概述

### 1.1 問題陳述

知識庫持續累積筆記（目前 150+ 篇），部分內容為非知識性質（stub、任務日誌、模板殘留）或知識不完整（缺結構、缺引用、深度不足）。缺乏系統化的品質篩選機制，導致低品質內容稀釋搜尋結果與知識價值。

### 1.2 目標

建立自動化內容評分系統，能：

1. **識別非知識內容**：stub、meta 模板、任務/日誌型、純連結、極短/空白、重複樣板
2. **識別不完整內容**：結構缺失、資訊密度低、缺引用、深度不足、結尾不完整
3. **產出可操作報告**：每筆筆記評分 + 等級 + 建議動作
4. **整合現有架構**：擴充 kb-curator Skill，沿用 groq-relay LLM 基礎設施

### 1.3 非目標

- 不自動刪除任何筆記（僅建議）
- 不取代 kb-curator 既有的模組 A-D
- 不引入新的外部服務或 API

## 3. 功能需求

### FR-1：規則型評分引擎（全量）

- 支援 2 個維度 × 6 個子項 = 12 個評分因子
- 維度 A（非知識訊號）權重 50%，維度 B（完整性訊號）權重 50%
- 每個子項 0-5 分，維度分數 = Σ(子項分) / 30 × 50
- 總分 = 維度 A 分數 + 維度 B 分數（0-100）

### FR-2：訊息熵分析

- 計算字元熵 H（Shannon entropy）
- 計算冗餘度 R = 1 - H / H_max
- 正常區間 H ∈ [7, 11]，R < 0.5
- 文本 < 50 字不計算（樣本不足標記 N/A）

### FR-3：LLM 語義評分（抽樣）

- 僅對篩選後子集呼叫（total_heuristic_score < 60 或 non_knowledge_flags >= 2）
- 透過 groq-relay `kb_score` mode（JSON 回傳）
- 速率控制 13 秒/筆，單次最多 20 筆
- 降級：LLM 不可用時使用關鍵字近似

### FR-4：報告產出

- 單筆評分 JSON 含：note_id, title, total_score, grade, dimensions, entropy, llm, action
- 彙總報告含：分布統計、候選清單、異常列表、建議動作
- 輸出至 `results/kb-content-score-report.json`

### FR-5：配置外部化

- 所有可調參數定義在 `config/kb-content-scoring.yaml`
- 含維度定義、熵參數、等級門檻、校準 hard caps、LLM 配置

## 5. 評分維度詳細規格

### 5.1 維度 A：非知識訊號（Non-Knowledge Signals）

| 子項 | ID | 滿分(5) | 零分(0) | 檢查方式 |
|------|----|---------|---------|----------|
| Stub/佔位符 | A.1 | 無 TODO/TBD/待補充 | 命中正則 | `TODO\|TBD\|待補充\|待續\|\.\.\.\|___` |
| Meta/模板語言 | A.2 | 純知識敘述 | 含指令語 | `請填寫\|請補充\|本模板\|\{\{.*\}\}` |
| 任務/日誌型 | A.3 | 非流程紀錄 | 含流程語 | `完成時間\|執行結果\|步驟\s*\d\|status:` |
| 純連結無脈絡 | A.4 | 有說明文字 | 連結多正文少 | `len(text)-len(urls)*20 < 50 && urls > 3` |
| 空/極短 | A.5 | ≥ 200 字 | < 30 字 | 字數分級 |
| 重複/樣板 | A.6 | 獨特內容 | 相似度 > 0.90 | difflib SequenceMatcher |

### 5.2 維度 B：完整性訊號（Completeness Signals）

| 子項 | ID | 滿分(5) | 零分(0) | 檢查方式 |
|------|----|---------|---------|----------|
| 結構完整性 | B.1 | 有標題+列表 | 無結構 | `##` 數 + 列表行數 |
| 資訊密度 | B.2 | 段落充實+熵正常 | 空洞/重複 | 段落分析 + 字元熵 + 冗餘度 |
| 概念覆蓋 | B.3 | 有定義/解釋 | 僅列點 | 定義性關鍵字 + LLM(抽樣) |
| 來源/引用 | B.4 | 有參考來源 | 無引用 | `\[1\]\|參考\|來源\|http\|\[\[` |
| 深度層級 | B.5 | 有深度說明 | 僅概念 | 五階段關鍵字 + LLM(抽樣) |
| 結尾完整性 | B.6 | 有結論 | 以待續結尾 | `待續\|\.\.\.$\|\(未完\)$` |

### 5.3 訊息熵規格

- **字元熵** H = -Σ p(x_i) log2(p(x_i))
- **冗餘度** R = 1 - H / log2(|字元集|)
- 正常區間 H ∈ [7, 11]，R < 0.5
- H < 7 → 疑似重複/stub（B.2 扣分）
- H > 12 → 疑似亂碼/異常（B.2 扣分）
- 文本 < 50 字 → 不計算，標記 entropy: N/A

### 5.4 等級門檻

| 等級 | 最低分 | 標籤 | 建議動作 |
|------|--------|------|----------|
| S | 96 | 卓越 | 可作為範本 |
| A | 81 | 優秀 | 保留 |
| B | 66 | 良好 | 保留 |
| C | 51 | 及格 | 建議優化 |
| D | 36 | 不完整 | 需補強 |
| F | 0 | 非知識/待刪 | 優先檢視 |

### 5.5 校準 Hard Caps

- contentText < 30 字 → 總分上限 25
- 含 TODO/TBD/待補充 → 非知識維度 -15
- 無任何 ## 或列表 → 完整性維度上限 40
- 字元熵 < 5 → 非知識維度 -10

## 6. 技術設計

### 6.1 檔案變更

| 操作 | 路徑 | 說明 |
|------|------|------|
| 新建 | `config/kb-content-scoring.yaml` | 評分配置 |
| 修改 | `bot/groq-relay.js` | 新增 kb_score mode |
| 修改 | `config/llm-router.yaml` | 新增路由規則 |
| 修改 | `skills/kb-curator/SKILL.md` | 新增模組 E |
| 修改 | `skills/SKILL_INDEX.md` | 更新觸發詞 |

### 6.2 資料流

```
KB API → 取得 notes → 規則型評分(全量) → 篩選低分 → LLM 評分(抽樣)
                                                           ↓ (失敗)
                                                    關鍵字近似(降級)
                                                           ↓
                                                    合併 → 報告產出
```

### 6.3 LLM 整合

- 透過 groq-relay.js 的 `kb_score` mode
- 回傳 JSON：`{concept_count, depth_level, info_density, is_knowledge, issues}`
- 速率：13 秒/筆（< 5 req/min）
- 快取：groq-relay 既有 5 分鐘快取 + 報告層 24 小時快取
