# 法華經一心三觀的知識產品化策略

## 從知識庫到 Podcast 系列與多層次報告的系統化轉化框架

> **研究系列**：lotus-sutra（post-synthesis 延伸）
> **日期**：2026-03-08
> **基礎**：KB 17 筆 synthesis 筆記（19,500 字完整報告 + CBETA 原典 + 跨宗派比較 + Podcast 腳本 + 精華版報告 + 方法論文件）
> **研究問題**：如何將一心三觀的豐富知識庫內容，系統化轉化為 Podcast 系列和多層次報告？

---

## 一、摘要（200 字）

本文提出一套「知識產品化」框架，將知識庫中已完成的法華經一心三觀學術研究體系（12 章 19,500 字），系統化轉化為三個層次的知識產品：（1）5 集 Podcast 系列，每集 15-20 分鐘雙主持人對話；（2）三級報告——學術完整版（19,500 字）、通俗深度版（5,000 字）、社群精華版（800 字）；（3）自動化管線設計，從 KB 筆記評分→腳本生成→TTS 合成→R2 發佈的端到端流程。框架的核心理念是「一源多流」——同一知識源頭，依受眾需求分流為不同形式，最大化知識傳播效率。

### 4.2 學術完整版（19,500 字）── 現有報告優化方向

**現有資產**：KB 筆記 #19be31e8（16,460 字 + 引用格式）

**優化建議**：
1. **引用格式統一**：全面採用 Chicago Style，CBETA 引用格式標準化為 T[冊]n[編號]_p[頁碼][行碼]
2. **補充文獻回顧**：整合 42 篇論文書目，新增「近十年學術回顧」章節
3. **增加圖表**：四代傳承時間軸、三觀與三惑對治關係圖、跨宗派比較矩陣
4. **排版規範**：中英雙語摘要/關鍵詞、分章節標號、附錄 CBETA 引文對照表

### 4.3 通俗深度版（5,000 字）── 故事化改寫

**結構設計**：

```
開場故事（300 字）
  → 你有沒有過這樣的經驗：同一件事，不同人有完全不同的看法？

第一部分：一千八百年的接力賽（800 字）
  → 龍樹的靈光一閃 → 慧文的頓悟時刻 → 智顗的集大成

第二部分：三個觀看世界的鏡頭（1,200 字）
  → 空觀：不是什麼都沒有，是看見本質
  → 假觀：不是虛假，是看見可能性
  → 中觀：不是折衷，是看見全貌

第三部分：當古老智慧遇見現代生活（1,500 字）
  → 職場壓力的三觀解法
  → 親子溝通的假觀智慧
  → 情緒管理的空觀練習
  → 聖嚴法師的四句話與三觀對應

第四部分：為什麼這很重要？（700 字）
  → 與現代心理學（ACT）的會通
  → 三分鐘三觀練習引導

**寫作原則**：
- 每段不超過 150 字（適合手機閱讀）
- 每個概念先給比喻，再給定義
- 佛學術語首次出現時用括號解釋
- 引用原文限 3 處以內（十如是、三是偈、一空一切空）

### 4.4 社群精華版（800 字）── 金句式圖文

```
[KB 筆記評分] → [素材選擇] → [腳本生成] → [TTS 合成] → [音頻串接] → [R2 上傳] → [ntfy 通知]
    ↑                                                                                    ↓

```

**現有工具鏈**：
- `tools/score-kb-notes.py`：筆記評分（knowledge_depth + podcast_suit + recency 等維度）
- `tools/generate_podcast_audio.py`：Azure TTS 語音合成
- `tools/concat_audio.py`：音頻串接 + 正規化
- `config/media-pipeline.yaml`：聲音配置
- `config/podcast.yaml`：上傳配置
- `templates/auto-tasks/podcast-create.md`：9 步驟自動任務模板

### 5.2 系列化擴展設計

現有管線是**單集生成**（從 KB 隨機選 3 筆筆記）。系列化需要以下擴展：

        status: planned

        status: planned
        kb_sources: ["19be31e8"]
        chapters: [7, 8, 9, 10, 11, 12]
    continuity:
      opening_style: "承接上集 + 本集預告"
      closing_style: "本集重點 + 下集預告"
      recurring_elements: ["三分鐘三觀練習", "花朵比喻回扣"]
```

#### B. 章節感知的素材提取（改進方向）

1. **章節定位**：從 16,460 字報告中按 `#` 標題切割為 12 個章節
2. **章節→集數映射**：依 yaml 的 `chapters` 欄位提取對應段落
3. **跨集去重**：已入播客的章節標記為 `used`，避免重複

#### C. 系列感知的腳本生成流程

```
讀取 podcast-series.yaml → 找到下一個 status=planned 的集數
→ 提取該集的 kb_sources + chapters
→ 讀取上一集的結尾（previous_episode_ending）
→ 生成系列腳本（含承接語和預告語）
```
