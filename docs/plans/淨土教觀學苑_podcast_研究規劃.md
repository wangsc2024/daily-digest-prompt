# 實作計畫：淨土教觀學苑 Podcast 製作研究藍圖

## 概要
- **目標**：建立一套可直接套用於淨土教觀學苑任一單集 Podcast 製作的標準研究包，涵蓋題目映射、來源策略、驗收規格、章節大綱與交接欄位。
- **預估任務數**：6 個
- **相關文件**：`C:\Users\user\.cursor\skills\writing-plans\SKILL.md`、`D:\Source\daily-digest-prompt\docs\plans\淨土教觀學苑podcast專輯.md`、`D:\Source\daily-digest-prompt\prompts\team\jiaoguang-podcast-one-episode.md`

## 架構說明
本研究規劃以 `docs/plans/淨土教觀學苑podcast專輯.md` 的 750 集題目索引為選題母表，將每一集先映射到「類別、課程、卷次、核心教義、目標聽眾、可用來源」，再依三層來源流程展開研究。第一層為淨土教觀學苑官網課程頁、講義、講記等原始材料；第二層為專案知識庫與內部規劃文件，用來承接既有佛學研究與節目機制；第三層為學術資料庫與公開平台，用來補強教義詮釋、歷史背景、音頻敘事與受眾理解。

本藍圖的產出單位不是「直接可錄製的腳本」，而是「單集研究包」。研究包完成後，後續腳本撰寫、雙主持人口語改寫、TTS、音檔合成、上架 metadata 填寫，都只需依本文件欄位逐步執行，不需再回頭查其他規範文件。

### 4. 來源採信順序
1. 官方課程頁、講義、講記、原典經論
2. 祖師著作、權威佛學教材、具明確作者之教學資料
3. 本機知識庫既有研究、專案內規劃與歷史製作紀錄
4. 學術論文、學位論文、專書導論
5. 官方影音與平台節目結構參考
6. 一般部落格與二手整理文章

### 5. 衝突處理規則
1. 若官方講義與二手整理不一致，以官方講義或講記為準。
2. 若不同學術文獻詮釋有差異，保留差異並標記學派或作者立場，不擅自合併成單一定論。
3. 若知識庫既有筆記與新查資料矛盾，須回溯原始來源後再更新研究包。
4. 若找不到足夠原始資料，該段落只能作「背景補充」，不得升格為節目核心主張。

## 單集研究包標準欄位
1. 基本識別：集數、題目名稱、課程名稱、卷次、類別、產出日期、版本號。
2. 研究定位：核心概念、研究問題、目標聽眾、節目目標、關聯前後集。
3. 來源表：官方原始資料、KB／內部文件、學術補證、公開平台參考，各自附連結或檔名。
4. 核心內容：核心論點 3 則、必講經論 2-3 則、祖師觀點 1-2 則、常見誤解 1-2 則。
5. 口語轉譯：術語白話表、生活化比喻、避免說法、適合主持人追問的轉場句。
6. 章節提要：開場、背景、核心教義、實修意義、今日行動、收束與 CTA。
7. 風險控管：來源衝突、資料不足、冷卻期重複、需人工複核處。
8. 後續交接：腳本摘要、節目標題候選、關鍵字、平台簡介、後製提示。

## 單集研究包完成定義
1. 題目已映射到題目池中的唯一列，並確認課程與卷次。
2. 已取得至少 1 份官方原始資料、1 則 KB／內部資料、3 份學術或權威補證。
3. 已完成 6-8 段章節提要與 1800-2400 字的內容密度估算。
4. 已完成術語白話表、重點論點、常見誤解與 CTA。
5. 已比對 `context/podcast-history.json` 與版本號，完成去重與回溯標記。
