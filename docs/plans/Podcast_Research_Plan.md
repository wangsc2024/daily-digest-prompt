# 研究計畫：淨土教觀學苑 podcast 製作

## 概要
- **目標**：建立一套可批次執行的淨土教觀學苑 podcast 單集研究包規格，涵蓋選題、資料蒐集、交叉驗證、章節設計與完成判準，供後續採訪、腳本、錄製與剪輯流程直接使用。
- **預估研究模組數**：5 個
- **相關文件**：
  - `C:\Users\user\.cursor\skills\writing-plans\SKILL.md`
  - `D:\Source\daily-digest-prompt\docs\plans\淨土教觀學苑podcast專輯.md`
  - `D:\Source\daily-digest-prompt\context\research-series.json`
  - `D:\Source\daily-digest-prompt\context\research-registry.json`

## 架構說明
本計畫依 `writing-plans` 的「概要 -> 架構說明 -> 技術棧 -> 可執行模組」框架改寫為研究型藍圖，核心不是直接生成 podcast 腳本，而是先建立「單集研究包」。研究包的最小完整單位為一集 podcast，必須同時具備：題目定位、課程對應、原始講義或講記依據、學術或典籍支撐、受眾設定、節目章節設計、爭議點註記與缺口處理。

研究流程採三層來源架構：
1. 第一層為淨土教觀學苑官方課程與講記，負責確認主題原義、課程脈絡與講次定位。
2. 第二層為佛學典籍與學術資料庫，負責校對教義、歷史脈絡、術語使用與不同詮釋。
3. 第三層為 podcast 與影音平台，負責觀察受眾語言、音頻節奏、敘事密度與節目包裝方式。

本研究已知前提如下：
- 專案本地文件已整理五大類、35 門課、750 集題目索引，可作為節目池主索引。
- `research-series.json` 已建立 `pureland-jiaoguan-podcast-planning` 系列，現階段需求集中在 application 前的標準化規格。
- 知識庫中已有「淨土教觀學苑 Podcast 製作研究規劃」與補強版內容，可作為基底，但仍需補完單集研究包的驗收與章節模板。

## 技術棧
- **研究主題領域**：淨土宗、天台教觀、佛典詮釋、佛教教育傳播、中文 podcast 製作
- **主要資料形式**：官網課程頁、講義、講記、佛典全文、學術論文、碩博士論文、影音節目、專案 KB 筆記
- **研究工具**：
  - 本地知識庫：`knowledge-query`（實測 `http://localhost:3000/api/health` 正常）
  - 專案脈絡：`research-series.json`、`research-registry.json`
  - 線上研究：官方網站、CBETA、Airiti Library、Google Scholar、國家圖書館博碩士論文系統、Podcast/YouTube 平台
- **輸出格式**：Markdown 研究包規劃文件，可延伸成每集 `brief.md` 或 JSON schema

## 執行方式
後續可依本文件逐章拆成單集研究包模板、來源採信規則與章節骨架，再交由 `executing-plans` 或其他製作流程執行。

## Task 2：核心議題清單

### 目標
定義單集研究包必須處理的核心子議題，避免後續只做資料堆疊而缺少可執行結構。

### 核心議題清單
1. **節目定位與受眾分層**：區分新學入門、淨土必修、止觀精修等課程層級，定義每集對應的聽眾前備知識。
2. **課程脈絡與講次映射**：確認單集題目對應的課程名稱、卷次、系列位置、前後集關聯與是否屬於導讀、講記、期中座談或補充講。
3. **教義核心與關鍵術語**：抽取每集必講的佛學核心概念、專有名詞、經論依據與必要定義。
4. **歷史傳承與祖師詮釋差異**：辨識該主題涉及的祖師、宗派、註疏傳統與可能的詮釋歧見。
5. **修行實踐與信眾日常連結**：把教理轉成修學、念佛、止觀、戒行或臨終關懷等可落地的生活語境。
6. **敘事結構與音頻節奏設計**：規劃單集章節、開場鉤子、主體段落、案例段與結尾收束，避免變成逐字講義朗讀。
7. **資料來源分級與採信規則**：建立官方課程、佛典、學術文獻、知識庫、影音資料的優先級與衝突解法。
8. **同題材內容去重與系列化策略**：處理 750 集中可能出現的重複概念，確保跨集不重講、同系列能遞進。
9. **節目可製作性與缺口補完機制**：當講義不全、音檔缺失、講記版本衝突或學術來源不足時，定義替代方案與風險標記。

### 各議題的驗收條件
| 子議題 | 研究充分的判斷標準 |
| --- | --- |
| 節目定位與受眾分層 | 完成 1 份受眾分層表，至少區分入門、進階、深修 3 層；每層列出收聽動機、預備知識與避用術語各至少 3 項。 |
| 課程脈絡與講次映射 | 每個研究包都能回填課程名稱、卷次、所屬類別、前後集關聯；至少核對 1 份官方課程頁與 1 份本地 750 集索引。 |
| 教義核心與關鍵術語 | 每集至少整理 5 個核心術語，並為每個術語提供 1 條官方講義或講記依據、1 條佛典或學術補充。 |
| 歷史傳承與祖師詮釋差異 | 至少取得 2 種不同來源的註解脈絡，例如祖師註疏與現代學術說明；對重大分歧至少產出 1 則「詮釋差異註記」。 |
| 修行實踐與信眾日常連結 | 每集至少整理 3 個生活化應用場景，並對應原課程內容，不得純憑想像發揮。 |
| 敘事結構與音頻節奏設計 | 每集完成 1 份 5 段式章節骨架，含開場、背景、核心義理、實修連結、結語；估計總時長需可落在 18 至 35 分鐘。 |
| 資料來源分級與採信規則 | 完成 1 份來源分級表；每集至少使用 1 個 A 級官方來源與 2 個 B 級以上補充來源；若僅有單一來源，必須標記風險。 |
| 同題材內容去重與系列化策略 | 對任一候選單集，至少檢查前後 3 集與同課程其他講次是否重複；列出 1 份「本集新增價值」摘要，至少 80 字。 |
| 節目可製作性與缺口補完機制 | 為每集填寫缺口檢查表；若任一核心欄位缺失，需列出替代來源、補查責任與是否允許延後製作。 |

3. C 級：YouTube、Podcast 平台節目頁、具署名的佛學部落格或寺院文章。
4. D 級：匿名論壇、未標示作者與出處的轉貼文，不得作為單獨依據。

### 搜尋關鍵字模板
